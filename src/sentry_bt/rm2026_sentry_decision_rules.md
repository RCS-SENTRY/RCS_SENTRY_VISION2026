## 1. 比赛与哨兵的定位

- RMUC 的核心形式是机器人之间的射击对抗；单局时长 **7 分钟**，目标是通过战术对抗击毁对方基地。（p.15）
- 正式阵容中每队可有：
  - 英雄 1
  - 工程 1
  - 步兵 2
  - 空中 1
  - **哨兵 1**
  - 飞镖系统 1
  - 雷达 1（p.22）
- 每场比赛首局最低上场阵容：**除雷达和飞镖系统外，至少 4 台机器人**。（p.22）
- 哨兵没有专属操作手名额，操作手表中**哨兵机器人对应人数为 0**；哨兵在规则上定位为自动/半自动执行单元，而不是普通遥操作地面机器人。（p.23）

## 2. 哨兵机器人基础规则总表

### 2.1 基本属性

- 机器人编号：**7**
- 初始区域：**启动区**
- 运行方式：**最多配置 1 个遥控器用于调试**
- 发射机构：**1 个 17mm 发射机构**
- 射击初速度上限：**25 m/s**
- 多机通信：
  - **可以向所有己方机器人发送数据**
  - **但只能接收雷达的数据**（p.29）

### 2.2 血量/热量/功率（与操作方式绑定）

哨兵**不适用经验与性能体系**，属性由“自动 / 半自动”模式决定。（p.30, p.117-118）

#### 自动哨兵
- 初始血量 / 上限血量：**400**
- 射击热量上限：**260**
- 射击热量冷却速率：**30 / s**
- 底盘功率上限：**100 W**（p.30）

#### 半自动哨兵
- 初始血量 / 上限血量：**200**
- 射击热量上限：**100**
- 射击热量冷却速率：**10 / s**
- 底盘功率上限：**60 W**（p.30）

### 2.3 初始允许发弹量与补弹方式

- 三分钟准备阶段可预装 17mm 和 42mm 弹丸
- 但哨兵实际初始允许发弹量为：**300 发**
- 七分钟比赛阶段可通过以下方式获取 17mm 允许发弹量：
  - 补给区、基地增益点、前哨站增益点兑换
  - **直接获取（哨兵专属补给区获取）**
  - 远程兑换（p.29, p.82-83）

### 2.4 哨兵可占领增益点

哨兵可占领以下增益点：（p.30）
- 基地增益点
- 中央高地增益点
- 梯形高地增益点
- 地形跨越增益点（公路 / 高地 / 飞坡 / 隧道）
- 前哨站增益点
- 补给区增益点
- 堡垒增益点

## 3. 决策系统必须理解的“比赛状态”

这些状态会直接决定哨兵能不能射击、能不能移动、能不能回血、能不能复活、是否可购买资源。

### 3.1 存活 / 脱战 / 战亡 / 异常离线（p.18-19）

#### 存活
- 裁判系统主控模块正常连入服务器
- 且血量不为 0

#### 脱战
- 在“存活”状态下，**连续 6 秒未发射弹丸且未被扣血**
- 比赛开始时默认处于脱战状态

> 脱战是哨兵远程兑换发弹量、远程回血等行为的重要前提条件。

#### 战亡
- 血量为 0
- 原因可能包括：
  - 装甲模块被攻击
  - 受撞击
  - 裁判系统模块离线
  - 被罚下
  - 因飞镖导致扣血（p.18）

#### 临时激活
- 战亡后，底盘和云台被临时上电
- **发射机构仍断电**（p.18）

#### 异常离线
- 比赛过程中主控模块断电或其他原因导致无法连入服务器（p.18）

## 4. 比赛流程中，哨兵在哪些时刻能做什么

### 4.1 三分钟准备阶段（p.129-131）

- 机器人放入各自初始区域
- 检查裁判系统
- 预装弹丸
- 调试机器人
- **三分钟准备阶段剩余 30 秒时，战场内所有机器人需上电**
- **剩余 15 秒时仍未上电的机器人需搬离战场**
- 三分钟准备阶段结束后：
  - **哨兵机器人、雷达的遥控器必须放到指定区域**
  - 不能继续持有并用于比赛（p.130）

### 4.2 哨兵模式选择窗口（p.117）

- **三分钟准备阶段开始后**
- **到十五秒裁判系统自检阶段前**
- 云台手可选择哨兵操作方式为：
  - 自动控制
  - 半自动控制
- 若未选择，则七分钟比赛阶段开始后**默认自动控制**

### 4.3 十五秒裁判系统自检阶段（p.131）

- 服务器自动检测：
  - 选手端连接状态
  - 各机器人模块状态
  - 场地道具状态
- 并**将所有机器人血量重置为满血**

### 4.4 五秒倒计时阶段（p.132）

- 选手端**不响应控制指令**
- 倒计时结束后比赛立即开始

### 4.5 七分钟比赛阶段（p.132）

- 哨兵正式进入对抗
- 所有真正的决策逻辑都在此阶段执行

## 5. 哨兵决策输入：必须感知或维护的关键变量

建议在决策系统中显式维护以下变量，它们都来自规则本身：

### 5.1 自身变量
- 当前血量 / 上限血量（p.19, p.30）
- 当前允许发弹量（p.19, p.82-83）
- 当前射击热量 `Q1`（p.19, p.71）
- 射击热量上限 `Q0`（与自动/半自动模式有关，p.30）
- 当前底盘功率上限（基础值受模式与姿态影响，p.30, p.117-118）
- 当前底盘能量（p.124）
- 当前姿态：进攻 / 防御 / 移动（p.117-118）
- 姿态切换冷却：**5 秒**（p.117）
- 各姿态累计持续时间是否已超过 **3 分钟**（p.117）
- 是否处于：
  - 存活
  - 脱战
  - 战亡
  - 异常离线
  - 虚弱
  - 无敌
  - 发射机构锁定（p.18-19, p.71, p.78-79）

### 5.2 战场变量
- 是否在补给区 / 基地增益点 / 前哨站增益点 / 堡垒增益点 / 高地等（p.109-114）
- 是否触发地形跨越增益（p.111-113）
- 己方前哨站是否存活（影响基地无敌、堡垒生效、前哨站占领等，p.103, p.113）
- 己方基地当前血量（影响复活加速、堡垒收益，p.79, p.113）
- 比赛剩余时间（影响金币、远程回血价格、立即复活价格、复活读条长度等，p.80-81, p.78）

### 5.3 联动变量
- 雷达发来的信息（哨兵**只能接收雷达数据**，p.29）
- 机器人是否被对方雷达标记到 `P>=100` 或 `P>=120` 导致易伤（p.120-122）
- 大/小能量机关是否可触发、是否正在激活、是否已激活（p.104-108）
- 己方经济、总经济、对方增益点占领状态等（信息波可提供，p.123-124）

## 6. 哨兵可执行动作清单

### 6.1 基础动作
- 移动
- 瞄准 / 发射 17mm 弹丸
- 占领增益点
- 进入 / 离开补给区
- 进入 / 离开堡垒
- 进入 / 离开中央高地 / 梯形高地 / 公路 / 隧道 / 飞坡等区域

### 6.2 通过裁判系统服务器主动执行的动作（哨兵自主能力，p.117）

哨兵可通过向裁判系统服务器发送信息，自主完成：
- 兑换允许发弹量
- 远程兑换发弹量
- 远程兑换血量
- 确认复活
- 兑换立即复活
- 切换姿态

### 6.3 云台手可对哨兵下发的干预动作（p.117）

云台手可通过裁判系统选手端干预哨兵：
- 转发小地图标记
- 在补给区 / 基地增益点 / 前哨站增益点购买允许发弹量
- 确认复活
- 远程兑换发弹量
- 远程兑换血量
- 远程兑换立即复活
- 切换姿态

### 6.4 自动 / 半自动与金币消耗差异（p.117）

- **半自动哨兵**：上述操作**不消耗金币**
- **自动哨兵**：云台手每执行一次上述操作，**额外花费 50 金币**

> 这意味着：
> - 自动哨兵更强，但“人工干预”更贵；
> - 半自动哨兵弱很多，但人工接管成本更低。

### 6.5 能量机关触发动作（p.104-105）

- 小能量机关、 大能量机关都可以由：
  - 步兵操作手通过选手端触发
  - **哨兵机器人通过裁判系统串口触发**
- 触发条件：该方当前有对应“激活机会”，且能量机关处于未激活状态

## 7. 射击相关规则：哨兵开火前必须满足的约束

### 7.1 有效伤害与基础伤害（p.69-70）

#### 17mm 对装甲有效检测速度
- 大装甲 / 小装甲 / 基地装甲 / 前哨站装甲：**大于 12 m/s**

#### 17mm 基础伤害
- 机器人装甲模块：**20**
- 基地装甲模块：
  - 上方前装甲模块：**5**
  - 其余 5 块装甲模块：**20**
- 前哨站中部装甲模块：**20**（p.70）

#### 注意
- 伤害检测看的是**接触装甲受攻击面的法向速度分量**
- 不是单纯看射击初速度（p.69）

### 7.2 射击初速度超限（p.70-71）

哨兵属于适用对象。设射速上限 `V0=25m/s`，实际射速 `V1`：

- `0 < V1 - V0 < 5`：锁定 **15 秒**
- `5 <= V1 - V0 < 10`：锁定 **20 秒**
- `10 <= V1 - V0`：**本局后续全部时间锁定，不再解锁**（p.71）

### 7.3 射击热量超限（p.71-72）

#### 热量增长
- 每检测到 1 发 **17mm**：`Q1 += 10`
- 每检测到 1 发 **42mm**：`Q1 += 100`
- 裁判系统以 **10Hz** 结算冷却（p.71）

#### 过热逻辑
设热量上限为 `Q0`：
- 对 17mm 发射机构：`Q2 = Q0 + 100`

当：
- `Q2 > Q1 > Q0`
  - 发射机构锁定
  - 第一视角图传断开
  - 直到 `Q1 = 0` 才恢复（p.71）
- `Q1 > Q2`
  - **本局后续全部时间锁定，不再解锁**（p.71）

#### 哨兵不同模式下的基础热量参数
- 自动哨兵：`Q0=260`，冷却 `30/s`
- 半自动哨兵：`Q0=100`，冷却 `10/s`（p.30）

## 8. 底盘功率与底盘能量：决定“机动性”的硬约束

### 8.1 底盘功率超限（p.72-73）

- 地面机器人底盘功率超限时先扣**缓冲能量**
- 缓冲能量耗尽后仍超限：
  - **底盘断电 5 秒**
- 检测结算频率：**10Hz**
- 缓冲能量上限 `Q = 60J`（p.72）

### 8.2 哨兵基础功率上限（p.30）
- 自动：100W
- 半自动：60W

### 8.3 姿态对底盘功率上限的影响（p.117-118）

#### 进攻姿态
- 底盘功率降为 **1/2**

#### 防御姿态
- 底盘功率降为 **1/2**

#### 移动姿态
- 默认效果：底盘功率上限提高为 **1.5 倍**
- 姿态衰减后：提高为 **1.2 倍**（p.118）

### 8.4 底盘能量机制（p.124）

每局：
- 英雄 / 步兵 / 哨兵初始底盘能量：**20000**
- 底盘能量上限：**40000**

#### 消耗
- 电源管理模块 `Chassis` 接口每输出 **1J**
- 底盘能量减少 **1**

#### 低能量惩罚
- 底盘能量降至 **0**
- 进入“节能”状态
- 底盘功率上限基础值变为 **35W**

#### 高能量奖励
- 底盘能量 `>=25000`
- 底盘功率上限基础值变为**对应基础值的 125%**
- 最高不超过 **200W**

#### 补能方式
- 占领补给区增益点时，可通过无线充电补充
- 当超级电容管理模块接口输入正在向超级电容传能时：
  - `超级电容输入能量 - Chassis 输出能量` 每达到 **1J**
  - 底盘能量增加 **8**（p.124）

## 9. 模块离线：哨兵决策需要紧急容错的事件

### 9.1 主控异常离线（p.73-74）

地面机器人异常离线期间：
- 发射机构、云台、底盘断电
- **每秒扣除上限血量的 5% 直到为 0**
- 场地交互模块失效
- 不再检测撞击 / 弹丸击打伤害 / 模块离线扣血
- 复活读条不再增加
- 多机通信断开（p.74）

### 9.2 装甲模块或超级电容管理模块离线（p.75）

- 每 500ms 检测一次
- 统计离线模块数 `N`
- 扣血值 `Value = 4 * N`
- 每个周期都会扣血（p.75）

### 9.3 测速模块离线（p.76）

- 若哨兵的 17mm 测速模块离线：
  - 对应 17mm 发射机构**立即锁定**
- 恢复连接后解锁（p.76）

### 9.4 定位模块离线（p.76）

若哨兵定位模块离线：
- **对方雷达对该机器人标记始终视为“准确”**
- **己方雷达对该机器人标记始终视为“错误”**（p.76）

> 这对哨兵非常致命，因为会直接放大对方雷达易伤收益，并削弱己方雷达协同。

## 10. 回血与复活：哨兵何时该撤、何时该买血、何时该复活

### 10.1 回血机制（p.77）

#### 补给区回血
地面机器人占领己方补给区增益点：
- 基础回血：**每秒恢复上限血量的 10%**

比赛开始 **4 分钟后**：
- 若机器人处于**脱战**状态
- 且占领己方补给区增益点
- 则获得 **每秒上限血量 25%** 的回血增益
- 一旦不再脱战，该 25% 立即失效（p.77）

#### 远程兑换血量
适用于英雄 / 步兵 / **哨兵**
- 前提：**处于脱战状态**
- 花费金币
- **确认后 6 秒生效**
- 恢复**此时上限血量的 60%**
- 但不超过上限血量（p.77）

#### 特别注意
- 若在这 6 秒内战亡：
  - 兑换无效
  - 金币**不返还**（p.77）

### 10.2 复活机制（p.78-79）

机器人战亡后：
- 射击热量重置为 0
- 缓冲能量重置为上限（p.78）

#### 读条复活
所需读条长度与：
- 战亡时比赛剩余时长
- 累计兑换立即复活次数
有关（p.78）

公式：
```text
所需复活读条 = 10 + (420 - 比赛剩余时长) / 10 + 20 × 累计兑换立即复活次数
```

- 单位：秒
- 小数四舍五入取整（p.78）

#### 读条速度
- 战亡后立即开始读条
- 默认：**每秒 +1**
- 若机器人位于补给区增益点，或己方基地血量低于 2000：
  - **每秒 +4**（p.79）

#### 读条复活完成后的状态
- 保留等级与经验值
- 无敌 **30 秒**
- 血量恢复到上限血量 **10%**
- 进入**虚弱**状态（p.79）

#### 虚弱状态效果
- 发射机构锁定
- 无法占领任何增益点
- 无法重建前哨站（p.79）

#### 解除虚弱的方法
在复活后，只要检测到可占领的：
- 前哨站增益点
- 基地增益点
- 补给区增益点  
对应场地交互模块卡后，虚弱解除。（p.79）

无敌结束规则：
- 若本次复活带来的无敌累计已超过 10 秒，则虚弱解除时无敌同步解除
- 若未满 10 秒，则补足到 10 秒后解除（p.79）

### 10.3 立即复活（p.79, p.81）

- 可在复活读条过程中使用金币兑换
- 效果：
  - 保留等级与经验
  - 无敌 **3 秒**
  - 血量恢复到 **100%**
  - 发射机构立即解锁
  - 底盘功率上限提高 **1 倍**（但不超过 200W），持续 **4 秒**（p.79）

## 11. 经济与允许发弹量：哨兵的资源决策核心

### 11.1 定时金币（p.79-80）

每队金币增量：
- 06:59：**400 初始金币**
- 05:59：+50
- 04:59：+50
- 03:59：+50
- 02:59：+50
- 01:59：+50
- 00:59：+150（p.80）

### 11.2 兑换价格（p.81）

#### 17mm 允许发弹量
- 远程兑换：**150 金币 / 100 发**
- 非远程兑换：**10 金币 / 10 发**
- 兑换上限：**1000 发 / 队**（p.81）

#### 血量（远程兑换）
```text
50 + ROUNDUP((420 - 比赛剩余时长) / 60 × 20) 金币 / 1 次
```
（p.81）

#### 立即复活

#### 占领己方堡垒收益
定义：
```text
Δ = 基地血量上限 - 当前基地血量
```

占领者获得：
- **50% 防御增益**

- “储备允许发弹量”上限：
```text
N = 100 + 2 * floor(Δ / 15)
```
- `N` 最大为 **500**
- 发射时优先消耗堡垒储备弹量，再消耗自身允许发弹量（p.113）

前提：
- 比赛进行 **3 分钟后**
- 且对方前哨站被击毁

效果：
- 占领中的机器人获得 **100% 易伤**
- 若单次占领累计满 **20 秒**
  - 对方基地护甲展开
- 计时在机器人死亡或占领失效后可保留 **3 秒**
- 期间计时暂停（p.114）

### 16.2 对方雷达标记带来的易伤（p.120-122）

- 范围：`0 ~ 150`

对**对方地面机器人**：

#### 重要后果
- 哨兵一旦被标记到 100 或 120 以上
- 实际承伤会明显变差
- 在高火力环境中应考虑更换姿态 / 退回防守点 / 进补给区 / 借地形跨越增益保命

### 16.3 双倍易伤（p.122）

- 雷达每累计使对方机器人易伤 **1 分钟**
- 获得 1 次触发“双倍易伤”的机会
- 可主动发送命令
- 使当前所有来源于雷达的易伤数值**翻倍**
- 持续 **30 秒**
- 每局至多 **2 次**（p.122）

> 这意味着对方雷达若触发双倍易伤，哨兵承伤风险会非常大。

### 16.4 信息波可提供的对方信息（p.123-124）

雷达解析信息波后，可获得对方：
- 各机器人位置
- 各机器人血量
- 各机器人剩余发弹量
- 各机器人当前持有的增益
- 队伍剩余经济
- 队伍总经济
- 对方各增益点被占领状态（p.123-124）

> 对哨兵决策而言，这些都是高价值输入。

## 17.2 姿态系统（p.117-118）

- 开局默认：**移动姿态**
- 切换冷却：**5 秒**
- 每局中，若累计处于某一姿态超过 **3 分钟**
  - 该姿态效果下降（p.117）

### 进攻姿态

#### 默认效果
- 获得 **3 倍热量冷却增益**
- 底盘功率降为 **1/2**
- 获得 **25% 易伤**

#### 衰减后
- 获得 **2 倍热量冷却增益**
- 底盘功率仍降为 **1/2**
- 仍获得 **25% 易伤**

### 防御姿态

#### 默认效果
- 获得 **50% 防御增益**
- 底盘功率降为 **1/2**
- 热量冷却速率降为 **1/3**

#### 衰减后
- 获得 **25% 防御增益**
- 底盘功率仍降为 **1/2**
- 热量冷却速率仍降为 **1/3**

### 移动姿态

#### 默认效果
- 底盘功率上限提高为 **1.5 倍**
- 获得 **25% 易伤**
- 热量冷却速率降为 **1/3**

#### 衰减后
- 底盘功率上限提高为 **1.2 倍**
- 获得 **25% 易伤**
- 热量冷却速率降为 **1/3**

### 姿态计算规则（非常关键，p.118）

- 上述计算均取整数值
- **热量冷却与底盘功率上限的变化，是在先完成其他增益计算后再乘算**
- 不是直接覆盖基础值

规则给出的示例：
- 自动哨兵基础冷却 30/s
- 大能量机关提供 3 倍冷却增益
- 堡垒再提供 +75/s
- 两者取较大者，先得到 `30 + 75 = 105`
- 再乘防御姿态衰减系数 `1/3`
- 最终实际冷却 `35/s`（p.118）

> 这说明姿态是“后乘算层”，实现时必须放在增益结算的后面。

## 19. 与哨兵实现强相关的“工程化状态机整理”

下面这部分不是规则原文，而是**按规则重组后的实现视角**，方便直接映射到行为树 / FSM / 决策层。

## 19.1 推荐一级状态

### A. `PREPARE`
- 放置于启动区
- 完成模式选择（自动 / 半自动）
- 预装弹丸
- 上电
- 等待自检（p.129-131, p.117）

### B. `MATCH_START_LOCK`
- 十五秒自检
- 五秒倒计时
- 不接受有效控制（p.131-132）

### C. `ALIVE_COMBAT`
- 存活且不处于特殊锁定
- 正常移动 / 射击 / 占点 / 买弹 / 切姿态

### D. `ALIVE_DISENGAGED`
- 存活且连续 6 秒未发弹、未被扣血
- 允许：
  - 远程兑换发弹量
  - 远程兑换血量（p.18, p.77, p.82）

### E. `SURVIVE_RECOVER`
- 血量低 / 热量高 / 弹量低
- 目标：
  - 去补给区
  - 去基地点 / 前哨站点买弹
  - 借地形跨越增益补生存
  - 根据基地失血决定是否抢堡垒（p.77, p.82-83, p.111-114)

### F. `POSTURE_ATTACK`
- 追求高冷却、高火力连续压制
- 风险：1/2 功率 + 25% 易伤（p.117）

### G. `POSTURE_DEFENSE`
- 追求高生存
- 风险：1/2 功率 + 冷却降为 1/3（p.118）

### H. `POSTURE_MOVE`
- 追求位移/转场/抢点
- 风险：25% 易伤 + 冷却降为 1/3（p.118）

### I. `WEAPON_LOCKED`
来源可能包括：
- 初速度超限
- 热量超限
- 测速模块离线
- 虚弱
- 战亡 / 异常离线（p.71, p.76, p.79）

### J. `DEAD_RESPAWN`
- 读条复活
- 或立即复活（p.78-79）

### K. `WEAK_AFTER_RESPAWN`
- 发射锁定
- 需触发补给区/基地点/前哨站点来解除（p.79）

### L. `PENALIZED`
- 黄牌底盘断电 / 血量扣除 / 界面遮挡
- 红牌直接罚下（p.134-135）

## 19.2 推荐关键触发条件

### 资源触发
- `ammo <= 阈值`：考虑去补给区 / 前哨站 / 基地点
- `脱战 && ammo不足`：考虑远程补弹
- `脱战 && hp低`：考虑远程回血
- `每分钟补给点次数累积`：考虑去补给区吃哨兵专属 +100 发（p.83）

### 生存触发
- `被雷达标记P >= 100`
- `基地血量差Δ变大`
- `热量接近上限`
- `底盘能量接近 0`
- `己方前哨站首次被击毁 -> 堡垒生效`
- `对方前哨站被击毁 && >3分钟 -> 可占对方堡垒`（p.103, p.113-114, p.120-122）

### 复活触发
- 战亡后：
  - 是否立即复活
  - 是否继续读条
  - 是否优先在补给区 / 基地点 / 前哨站点结束虚弱（p.78-79）

### 姿态触发
- 追击 / 输出：进攻姿态
- 守点 / 抗压：防御姿态
- 转场 / 抢点 / 过地形：移动姿态
- 单姿态累计 > 3 分钟：需要预留轮换策略，避免姿态衰减（p.117-118）

## 20. 最后给实现者的规则检查表

在开始写哨兵决策代码前，至少确认以下规则都被映射到了程序里：

- [ ] 自动 / 半自动模式选择时机正确
- [ ] 自动 / 半自动基础属性不同
- [ ] 哨兵只接收雷达数据
- [ ] 远程兑换发弹量 / 血量都要求脱战
- [ ] 远程兑换生效有 6 秒延迟
- [ ] 哨兵补给区每分钟 +100 发可累积
- [ ] 姿态切换冷却 5 秒
- [ ] 姿态累计超过 3 分钟会衰减
- [ ] 姿态对底盘功率 / 冷却是“后乘算”
- [ ] 热量超限与速度超限都可能永久锁枪到本局结束
- [ ] 模块离线要触发降级处理
- [ ] 复活后会进入虚弱，必须去特定点解除
- [ ] 堡垒生效依赖己方前哨站首次被击毁
- [ ] 对方堡垒推进依赖“比赛超过 3 分钟 + 对方前哨站被击毁”
- [ ] 禁区、冲撞、补给区阻挡等违规要纳入路径规划约束
- [ ] 装甲遮挡不能发生
- [ ] 黄牌对哨兵有专门的“底盘断电 2 秒”效果
